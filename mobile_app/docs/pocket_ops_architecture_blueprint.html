Pocket Ops - Blueprint do MVP Mobile

Estrutura inicial para migrar o dashboard financeiro atual para um app Flutter real, preservando o contrato do backend em AppScript e adaptando a experiência para leitura vertical, toque e execução local em Android e iPhone.

Sprint Complexa | v2.0.0

Manter

Contrato do payload, cálculo de score, radar, missão, recomendação principal e Esquilo IA.

Ajustar

Hierarquia visual, navegação, loading, componentes de detalhe por categoria e consumo HTTP.

Recriar

Shell mobile nativo, roteamento Flutter, cards táticos, estado local e endpoint JSON do AppScript.

Flutter App

App Shell main.dart + PocketOpsApp MaterialApp, tema tático leve, roteador simples e bootstrap por `dart-define`.
Tela 1 Home / Visão Patrimônio, performance consolidada, score, alocação por macroclasse, radar e missão.
Tela 2 Carteira / Categorias Blocos de Ações, Fundos e Previdência com navegação para detalhes por categoria.
Tela 3 Insights / Esquilo IA Quebra do score, recomendação principal e leitura remota sob demanda.
Fluxo: boot do app → carrega dashboard → monta home → abre detalhe → consulta IA quando necessário.

Estado + Serviços

Estado DashboardController Controla loading, erro, refresh, payload atual e leitura da Esquilo IA sem duplicar regra.
Serviço AppScriptDashboardService Encapsula chamadas HTTP para `dashboard`, `ai-analysis` e `health` com timeout e parse seguro.
Modelos DashboardPayload + CategorySnapshot + PortfolioHolding Normalizam o JSON do AppScript para widgets pequenos e páginas de detalhe.
UI Blocks Widgets táticos independentes Hero, anel de alocação, radar, missão, score e cards por ativo ficam isolados por responsabilidade.

AppScript API

Entrada Backend_Core.gs `doGet()` segue servindo HTML por padrão e passa a entregar JSON quando `format=json`.
Roteamento Mobile_Api.gs Resolve `resource=dashboard`, `resource=ai-analysis` e `resource=health` sem quebrar o web atual.
Contrato Payload reaproveitado `getDashboardData()` continua sendo a fonte principal. O mobile apenas consome via HTTP.
Segurança Token opcional por Script Property Se configurado, o endpoint mobile valida `MOBILE_APP_API_TOKEN` sem mudar regra de negócio.

Dados + Backend

Fonte primária BigQuery Tabelas operacionais seguem como leitura principal da carteira.
Fallback Google Sheets Planilha operacional continua servindo contingência, cache de mercado e histórico.
IA Gemini / OpenAI Mesma estratégia atual, sem lógica paralela para o app mobile.
Backbone: BigQuery → normalização Apps Script → payload unificado → endpoint JSON → Flutter.

Decisão de Migração

  • Manter: score, alertas, missão, radar, recomendação e payload central.
  • Ajustar: linguagem visual, navegação por tabs, detalhe de categoria e leitura mobile.
  • Recriar: app shell Flutter, componentes nativos, endpoint HTTP e setup local.

Caminho Seguro do MVP

  • 1. Entregar home enxuta e navegável.
  • 2. Consumir somente o contrato já existente do AppScript.
  • 3. Deixar Android pronto para teste local imediato.
  • 4. Preparar iOS estruturalmente, com execução dependente de Mac + Xcode.
